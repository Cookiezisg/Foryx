# Forgify 产品哲学（交互形态）

> 这里写「Forgify 长什么样、为什么这样」。它是**形态的事实源**——
> 先在 `design-lab/` 用 HTML 把交互聊清楚，定稿后再统一落 Flutter。
> 这些 demo **不连后端**，纯讲交互逻辑与视觉语言。

---

## 一、外壳（视觉参考：Claude Code 桌面端）

**方向（2026-06-14 定）**：主外壳以 **Claude Code 桌面端为视觉参考**——结构 / 布局 / 密度照它来，
但**颜色**（干净白/近黑，非 Claude 米白暖色）+ **字体**（系统字，非衬线）用我们自己的。结构上：

- **圆角浮窗 + 海与岛（同色，靠「海岸线」分）**：整窗是浮在桌面的圆角窗。
  **中间主区 = 海**（平铺，对话直接铺在海上、无卡）；**侧栏、右侧面板 = 岛**。
  关键：**海与岛同色**（浅色都白 `#fff`、暗色都深 `#1c1c1e`），**分割只靠「海岸线」**——
  1px 描边 + 12px 圆角 + 8px 窄缝 + 一丝微阴影，**不靠颜色差**（中间不发灰、岛也不另上色，干净白/深、非米白）。
  composer 的 git 条 / 输入框 / 用户气泡同理（同色 + 海岸线）。
- **红绿灯 / 收起 / 搜索在左岛顶**（在岛内，不浮在岛外的标题栏）；
  **面包屑（repo / session ⌄）+ 主题/面板切换在主区(海)顶**。
- **左岛顶三切换：抄 Notion 原理、风格用我们的**——选中项展开成「图标 + 文字」灰药丸、
  未选只图标，切换走 spring 缓动**丝滑过渡**（不抄 Notion 的样子，圆角/灰/缓动都用我们的）。
- **用户气泡是灰药丸**（`--cc-bubble`，**不走海岸线**）——这是「同色 + 海岸线」规则的唯一例外。
- **标题栏**：macOS 红绿灯 + 侧栏折叠/搜索 ｜ 面包屑（repo / session ⌄）+ 面板切换。
- **侧栏**：模式段（Chat / Tasks / Code）· New session · Routines · Dispatch[Beta]
  · Customize · More · **Recents 会话列**（小圆点状态 + 选中中性灰高亮）· 底部用户位。
- **对话**：无头像、AI 全幅平铺 markdown、用户右侧柔和气泡、每个 turn 前一个 ✳ sparkle 标记。
- **工具活动**：摘要行——`Used N tools ›` / `Edited <file> +n −m ›` / `Ran a command ›`
  / `Background shell completed …`（状态行无箭头），点开看 detail（左边线 mono 块）。
- **底部 composer**：git 上下文条（repo · branch · +n −m · Create PR）+ 输入框（/ 命令 + 回车）
  + 工具条（Auto · + · mic ｜ 模型 · 模式 · spinner）。
- **明暗双主题**。

> 先把视觉参考做扎实，确认味儿对了，再讨论把 Forgify 自己的 IA（四实体 / 信号交互）映射进这套外壳。

## 二、颜色 + 字体（这两样不取 Claude 的）

- **配色**：纯净白、近黑、系统灰阶（**非** Claude 米白暖色）。强调色是**单一 token**
  `--accent`（默认 Apple blue `#0071e3`，换品牌色 = 改一行）。
- **少强调色（重要）**：accent 极克制——只在「正在发生」处点极小一抹（如实体卡的「● 编辑中」小圆点）。
  实体图标 / 标签 / 面板一律中性灰，**绝不上大色块 / 大渐变**；语义色（成功/警告/危险）仅用于真实状态、不当装饰。
- **精修形态沉淀在** [`reference.html`](reference.html)（设计参考页：原则 + 配色 token + 实体卡两态 + 核心组件），供真实设计 / Flutter 落地直接参考。
- **字体**：系统字体栈，不用花字。语义清楚 > 个性。
- **明暗双主题**：亮 / 暗 / 跟随系统，全走 CSS 变量，一处定义全局切换。
- **紧凑**：行高基准 `32px`；`tool_call` 与 `reasoning` 默认折叠——信息密度高但不喧闹。
- **动效是灵魂**：收尾微弹的 spring 曲线（`--ease-spring`），让一切切换有「物理感」。

## 三、四导航（Chat · Forge · Documents · Notifications）

左导航**顶部**四个可切换项。**选中的展开显示名字 + 高亮 pill，未选的只剩一个图标**；
切换时旧标签收起、新标签弹开，高亮平滑移动（参考 Notion 2026.03 新版 UI 的切换动效——
*当前为近似实现，待对照原参考微调*）。导航**底部**是工作区信息 + 设置。

- **Chat**：主入口，与一般对话产品无异（见下「信号交互」是唯一的不同）。
- **Forge**：四类实体的家（**Quadrinity**：Function / Handler / Agent / Workflow）。
  *命名建议 `Forge`——契合品牌「锻造能力」；你定。* trigger / control / approval 不进顶层，
  它们是 workflow 图里的节点，活在 workflow 构建器内部。
- **Documents**：知识库（文档 / 记忆 / 技能）。
- **Notifications**：通知（运行完成、待审批、实体变更…），带未读角标。

## 四、信号交互（Streaming Entity Card）—— 灵魂

> **已在 `demos/main-shell.html` 落地**：AI 调 `update_agent` 时右侧实体卡（岛，同色 + 海岸线）
> 从右滑入，model 闪光改值、system prompt 逐字流入、`web_search` 工具弹入、版本 v4→v5；
> 中间对话同步出 `Used 2 tools ›` + `Updated Researcher v4→v5`；卡可关闭、ref 药丸点了重唤。

这是 Forgify 与普通 chat 产品**唯一也是最重要的不同**，要非常认真地做：

聊天里 AI 调用了和某个**实体**相关的 Tool Call（如 `update_agent`）时——
**那个实体的卡片从右侧作为一座岛滑入**，并以**流式效果**实时展示 AI 的编辑过程：
改动的字段闪光、系统提示逐字流入、新工具弹入。卡片可像侧栏一样**收起**为细条、
可**手动切换**看不同实体；但只要 AI 聊到了某个实体，就**动画切换**过去。

- **物理直觉**：左边是「正在发生的对话」，右边是「对话正在塑造的东西」。
  AI 不只是回字，它在你眼前**锻造实体**——这正是 Forgify 的世界观。
- **数据契约**：实体卡的流式来自后端三条 SSE 之一（`entities` / `messages`）。
  铁律 **DB 行是真相、流只为实时**：`seq>0` 才耐久落缓存 / 推进续传游标；
  `delta`·`tick`（`seq=0`）只改瞬时视图态、不进耐久层。
- **它也是通用面**：在 Forge 里点任一实体，同一张卡静态打开（无流式）——
  证明右岛是统一的「实体详情/编辑」面，chat 只是给它接上了 AI 的实时驱动。
- **工具调用块 = 复刻 Claude Code（不自创；规矩详见 [`reference.html`](reference.html) 四）**：
  平时**一行、无框**（`Used N tools ›`）；**展开**是一个外框，里面是工具列表；**点某个工具**展开它自己的
  详情框（命令 = 终端输出；编辑 = 绿增红删 diff，行距舒服）。只有一个工具时无外框、直接一行 + 展开详情。
  **实体类工具**（如 `update_agent`）执行时**唤出右侧实体卡**（见上）看全貌、流式编辑——对话内不再单独放「实体缩影」块。
  diff 的 ＋/− 用语义绿红、其余一律中性。

## 五、Onboarding（首启两步）

1. **外观 + 语言**：亮 / 暗 / 跟随系统；语言偏好。选主题**即时生效**，当场看到效果。
2. **API Key**：大模型 Key（必须）+ 联网搜索 Key（可选）。
   文案强调**只存本地 SQLite、绝不外传**——单进程单用户本地应用的信任基石。

配完进主屏。

## 六、和 Flutter 的关系

`design-lab/` 是**形态草图 + 交互事实源**，不是产品代码、不连后端、不进文档门禁。
形态在这里聊定后，统一在 `frontend/`（Flutter）落地——按
[`ADR 0004`](../docs/decisions/0004-frontend-flutter-architecture.md) 的三层 feature-first 架构，
用 Riverpod + freezed + SSE gateway 把这些交互接到真后端契约上。
