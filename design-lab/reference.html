Forgify · 设计参考

把 main-shell 精修后的形态沉淀于此，供后面真实设计 / Flutter 落地直接参考。配色 = 干净白/近黑（非米白）。

一 · 核心原则

海与岛，靠海岸线分

中间主区 = 海（平铺，对话直接铺其上）；侧栏 / 实体卡 / 输入框 = 岛。海与岛同一颜色，分割只靠 1px 描边 + 12px 圆角 + 8px 窄缝 + 微阴影——不靠颜色差。

少强调色

整体克制：仅在「正在发生」处点极小一抹 accent（如卡片的「● 编辑中」小圆点）。图标块、标签、面板一律中性灰，不上大色块。

结构参考 Claude Code，色与字用我们自己的

圆角浮窗、齐边面包屑、工具摘要行、composer 的结构以 Claude Code 为视觉参考；但底色用干净白/近黑（非米白）、字体用系统字（非衬线）。

切换抄 Notion 原理

选中项展开成「图标 + 文字」药丸、未选只图标、spring 丝滑过渡——只取原理，圆角/灰/缓动都用我们自己的。

二 · 配色 token（浅色；暗色对应翻深）

强调色是单一 token --accent（默认 Apple blue #0071e3，换品牌色 = 改一行）。语义色（成功 / 警告 / 危险）仅用于真实状态，绝不当装饰。
