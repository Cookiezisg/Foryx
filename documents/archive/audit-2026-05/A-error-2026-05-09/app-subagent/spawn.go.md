## Notes

- **§S9 critical-path identification**: The Spawn body has THREE potential terminal writes — sub-Message body persistence (in host.WriteFinalize, audited separately), Status reconcile (site #11, correctly detached), and placeholder BlockStop emit (site #13, NOT detached). The recent fix referenced in the brief was to chat/host.go's WriteFinalize which now uses detached ctx for both DB write and StopMessage emit (chat/host.go:54-77). subagent's WriteFinalize and Spawn need the analogous fix at site #13 + host.go's StopMessage to align.
- **Post-2026-05 schema convention**: All Sub-Message persistence flows through the unified `messages` + `message_blocks` tables via `chatRepo`. There's no longer a SubagentRun table to write — the DB row IS the sub-Message. This means the §S9 critical writes are exactly the chat ones, which makes the chat/host.go pattern directly transferable.
- **idgenpkg vs raw rand**: spawn.go uses `idgenpkg.New(prefix)` exclusively — no raw `crypto/rand.Read` calls. §S15 panic-on-rand-failure is enforced inside pkg/idgen.New, so this caller is compliant by delegation.
- **sentinel chain at site #1**: `fmt.Errorf("subagentapp.Spawn: %w: %q", ErrTypeNotFound, typeName)` — note the `%q` on typeName (not `%w`/`%v`) is standard Go for embedding a string literal with quotes for log readability. Doesn't break errors.Is unwrap because `%w` precedes it.
