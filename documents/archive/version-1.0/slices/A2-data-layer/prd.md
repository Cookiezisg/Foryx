## 1. 这块做什么

本地数据的存储和迁移基础。用户不直接感知这一层，但它的稳定性决定了所有数据是否安全。

---

## 2. 用户感知到的行为

### 2.1 首次启动

- 首次打开 Forgify，数据库自动在后台创建，用户无感知
- 无任何弹窗或进度条（速度足够快，不需要）

### 2.2 版本升级时的迁移

- 新版本 Forgify 启动时，如有 schema 变更，自动迁移
- 迁移失败时（极少发生）：弹窗告知用户，提供"查看错误"和"联系支持"选项，不自动删除数据

### 2.3 数据文件位置

用户可在设置页查看和修改数据存储路径：

| 系统 | 默认路径 |
|---|---|
| macOS | `~/Library/Application Support/Forgify/` |
| Windows | `%APPDATA%\Forgify\` |
| Linux | `~/.config/Forgify/` |

修改路径后，Forgify 询问是否迁移现有数据到新路径。

### 2.4 数据文件结构（用户可见）

```
Forgify/
├── forgify.db          # 主数据库
├── forgify.db-wal      # SQLite WAL 文件（正常现象，不要删）
└── tools/              # 工具代码文件（由工具系统管理）
```

## 3. 不在本切片范围内

- 数据备份 / 云同步（未来版本）
- 数据导出功能
- 具体业务表的内容（各切片自己定义并添加迁移文件）

---

## 4. 验收标准

- [ ] 首次启动自动创建数据库，无报错
- [ ] 数据库文件在正确的默认路径下
- [ ] 关闭再打开 Forgify，数据不丢失
- [ ] 模拟迁移失败场景，弹窗正常出现，数据不损坏
- [ ] 设置页可查看当前数据路径
