# E3 · 基础节点类型 — 产品需求文档

**切片**：E3  
**状态**：待 Review  
**依赖**：E1  
**下游**：F1

---

## 1. 触发节点 (Trigger)

工作流的起点，每个工作流有且只有一个触发节点。

| 触发类型 | 配置项 |
|---|---|
| **手动触发** | 无配置，点击"运行一次"触发 |
| **定时触发** | Cron 表达式（有可视化编辑器："每天 9:00"）|
| **Webhook** | 生成唯一 URL，接收 POST 请求触发 |
| **文件监控** | 监控指定目录，新文件出现时触发 |

节点展示：`⏰ 定时触发 · 每天 09:00`

---

## 2. 工具节点 (Tool)

调用一个工具（内置或用户锻造）。

配置面板：
- 选择工具（下拉列表，搜索）
- 为每个参数填写值或绑定上游节点的输出（`{{node_2.result.email}}`）
- 显示工具的输出 schema（方便下游节点绑定）

节点展示：`📦 读取 Gmail`  
缺失工具时显示：`⚠️ [缺失] 解析供应商A的Excel`（黄色）

---

## 3. 条件节点 (Condition)

根据条件决定走向，支持 if/else 和 switch。

```
[条件节点]
    ├── 条件 A 成立 → [分支 A]
    ├── 条件 B 成立 → [分支 B]
    └── 否则 → [默认分支]
```

配置：
- 条件表达式：`{{node_2.result.total}} > 10000`
- 支持多条件 AND/OR
- 每个分支有独立的出线

---

## 4. 变量节点 (Variable)

定义工作流级别的常量或中间变量，供其他节点引用。

配置：
- 变量名（`sender_email`）
- 类型（string/number/boolean）
- 值（静态值或表达式）

---

## 5. 人工确认节点 (Approval)

暂停工作流，向 Inbox 发送审批请求，等待用户操作。

配置：
- 审批标题（如"确认发送邮件"）
- 展示给用户的信息（可引用上游数据）
- 超时时间（默认 24 小时，超时后走"拒绝"分支）

用户在 Inbox 操作：批准 / 拒绝 / 修改参数后批准

---

## 6. 验收标准

- [ ] 触发节点可选择 4 种触发类型并正确配置
- [ ] 工具节点：选工具 → 参数支持绑定上游输出（`{{}}`语法）
- [ ] 缺失工具节点显示黄色警告
- [ ] 条件节点支持多分支，连线对应分支颜色区分
- [ ] 人工确认节点触发 Inbox 消息
