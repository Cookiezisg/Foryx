# F3 · Mailbox 审批 — 产品需求文档

**切片**：F3  
**状态**：待 Review

---

## 1. 背景

工作流运行时遇到需要人工确认的操作（Approval 节点），不应阻塞整个工作流。Mailbox 模式：将该操作放入 Inbox 等待用户审批，工作流的其他无依赖分支继续运行。

---

## 2. 核心概念

**Approval 节点** 是工作流设计者主动放置的节点，用于在敏感操作（如发邮件、调用付费 API）前暂停并等待用户确认。

与 H1（权限门控）的区别：
- Approval 节点 = 设计者主动放置，用于业务流程中的人工判断点
- H1 权限门控 = 系统自动检测执行级操作，强制拦截

---

## 3. 用户体验

### 审批通知

工作流执行到 Approval 节点时：
1. 该节点变为橙色（等待审批）
2. Inbox 出现一条新消息，侧边栏 Inbox 图标显示红点
3. 工作流中无依赖该节点的其他分支**继续运行**

### Inbox 中的审批消息

```
┌─────────────────────────────────────────────┐
│  ⏳ 需要你的确认                             │
│  工作流：发送周报邮件                         │
│  节点：发送邮件确认                           │
│                                             │
│  即将发送邮件至 boss@company.com             │
│  主题：本周工作汇报                           │
│                                             │
│  [拒绝]  [修改后批准]  [批准]                │
└─────────────────────────────────────────────┘
```

### 三种操作

| 操作 | 结果 |
|---|---|
| **批准** | 操作执行，工作流从该节点继续 |
| **拒绝** | 该节点标记为"已拒绝"，触发节点配置的"拒绝逻辑"（跳过/停止/通知） |
| **修改后批准** | 用户修改参数（如邮件收件人），修改后执行 |

---

## 4. 右侧上下文面板

点击 Inbox 中的审批消息，右半区显示关联工作流的 Canvas：
- 等待审批的节点高亮为橙色
- 整个执行路径（已成功的节点）显示绿色

---

## 5. 超时处理

Approval 节点默认等待 **24 小时**，超时后：
- 节点状态变为"超时"
- 按拒绝逻辑处理
- Inbox 消息更新为"已超时"

超时时长可在 Approval 节点配置中修改。

---

## 6. 验收测试

```
1. 工作流执行到 Approval 节点 → 节点变橙色 → Inbox 出现审批消息
2. 工作流其他分支（不依赖该 Approval 节点的）继续执行
3. 点击"批准" → 工作流从该节点继续 → 节点变绿
4. 点击"拒绝" → 节点标记拒绝 → 执行拒绝逻辑
5. 点击"修改后批准" → 弹出参数修改界面 → 修改后执行
6. 超时 24 小时 → 自动按拒绝逻辑处理
```
