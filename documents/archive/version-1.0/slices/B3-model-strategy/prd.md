## 1. 这块做什么

决定在不同场景下用哪个模型，以及主模型失败时如何自动降级。用户能感知到的是：AI 会用合适的模型做合适的事，失败时自动切换而不是直接报错。

---

## 2. 场景分类

Forgify 内部有四种不同的 LLM 使用场景，对模型能力要求不同：

| 场景 | 说明 | 推荐模型特性 |
|---|---|---|
| **主对话** | 用户和 AI 的日常对话 | 推理强、上下文长 |
| **代码生成** | 工具锻造时生成 Python 代码 | 代码能力强 |
| **低成本任务** | 上下文压缩摘要、autoDream 整理 | 便宜、快 |
| **工作流 LLM 节点** | 用户在工作流里配置的 LLM 节点 | 用户自选 |

---

## 3. 用户配置界面

在设置页"模型与 API" → "模型分配"区块：

```
主对话模型    [Claude Sonnet 4.6     ▼]
代码生成模型  [与主对话相同          ▼]（可独立配置）
低成本模型    [Claude Haiku 4.5      ▼]（自动用于内部任务）
备用模型      [GPT-4o-mini           ▼]（主模型失败时自动切换）
```

- 下拉选项来自已配置的 API Key 所支持的模型
- "与主对话相同"是默认选项，不需要单独配置
- 备用模型可以设为"无"（关闭自动降级）

---

## 4. 自动降级行为

| 情况 | 行为 |
|---|---|
| 主模型调用成功 | 正常使用，不提示 |
| 主模型失败，备用模型可用 | 静默切换，在本条消息下方显示小字："已自动切换到备用模型 GPT-4o-mini" |
| 主模型和备用模型都失败 | 显示错误，建议用户检查设置 |
| 主模型额度耗尽 | 直接切换备用，不重试 |

---

## 5. 不在本切片范围内

- 每次对话选择模型的 UI（交给 C1）
- 用量统计和费用估算
- 自动选择"最优"模型（用户永远是决策者）

---

## 6. 验收标准

- [ ] 设置页可配置四个场景的模型
- [ ] 主对话、代码生成、低成本场景分别调用正确的模型（可用日志验证）
- [ ] 主模型失败时自动切换备用，消息下方有小字提示
- [ ] 两个模型都失败时显示错误
- [ ] 配置持久化，重启后不丢失
