## 1. 这块做什么

每个工具的 Python 代码都在隔离的子进程中运行。
用户不需要安装 Python 或任何依赖，Forgify 内置了完整的 Python 运行环境。

---

## 2. 用户感知

用户感知不到沙箱的存在——点击"运行测试"，几秒后看到结果。
如果工具需要安装依赖，第一次运行时会显示"正在安装依赖..."的进度提示（通常 1-5 秒）。

---

## 3. 运行环境

### 3.1 Python 版本管理

Forgify 使用内置的 `uv` 工具（随应用打包）管理 Python 环境：
- 首次运行自动下载 Python 3.12（缓存在用户数据目录）
- 每个工具在独立的虚拟环境中运行
- 依赖安装结果缓存，相同依赖组合只安装一次

### 3.2 超时限制

- 默认超时：30 秒
- 超时后强制终止子进程
- 用户在工具设置中可调整（5-300 秒）

### 3.3 资源限制

- 内存软限制：512MB（超出时记录警告，不强制终止）
- 禁止创建子进程（防止逃逸）
- 不限制网络访问（工具可能需要调用外部 API）

---

## 4. 输入输出协议

工具调用时：
- 输入：JSON 格式参数通过 stdin 传入
- 输出：函数 return 值序列化为 JSON 通过 stdout 传出
- 错误：异常信息通过 stderr 传出

示例（内部运行的 wrapper 脚本，用户不可见）：
```python
# _runner.py（Forgify 内部生成）
import sys, json
from tool_code import send_email

params = json.load(sys.stdin)
try:
    result = send_email(**params)
    print(json.dumps(result))
except Exception as e:
    print(json.dumps({"error": str(e), "type": type(e).__name__}), file=sys.stderr)
    sys.exit(1)
```

---

## 5. 依赖管理

工具代码中的 `import` 语句会被静态分析，识别第三方库（非标准库）后自动安装：
- 安装命令：`uv pip install <packages>`
- 安装结果按依赖组合哈希缓存（避免重复安装）
- 如果安装失败，返回错误信息给用户

---

## 6. 错误类型

| 错误类型 | 用户提示 |
|---|---|
| 超时 | "工具运行超过 30 秒，已停止。可在工具设置中调整超时时间。" |
| 语法错误 | "代码存在语法错误：[具体位置和原因]" |
| 依赖安装失败 | "无法安装 [包名]：[错误原因]" |
| 运行时异常 | "[异常类型]: [异常消息]，发生在第 N 行" |
| 返回值格式错误 | "工具返回了非 JSON 格式的数据" |
