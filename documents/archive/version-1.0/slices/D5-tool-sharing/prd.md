## 1. 这块做什么

用户可以将工具导出为文件，分享给他人；也可以从文件导入别人分享的工具。
纯本地文件操作，不涉及云端。

---

## 2. 导出工具

### 2.1 触发方式

- 工具详情页右上角"导出"按钮
- 工具库卡片右键菜单"导出"（可选，后续迭代）

### 2.2 导出格式

导出为 `.forgify-tool` 文件（实质是 JSON），文件名为 `{工具名}.forgify-tool`：

```json
{
  "version": "1.0",
  "exported_at": "2026-04-19T10:00:00Z",
  "tool": {
    "name": "send_email",
    "display_name": "发送邮件",
    "description": "通过 SMTP 发送邮件到指定地址",
    "category": "email",
    "code": "def send_email(to: str, ...) -> dict:\n    ...",
    "requirements": ["requests"]
  }
}
```

`requirements` 由 Forgify 从 `import` 语句静态分析提取（只含第三方包）。

### 2.3 导出操作

点击"导出" → 弹出系统文件保存对话框 → 用户选择保存位置 → 完成。

---

## 3. 导入工具

### 3.1 触发方式

- 工具库右上角"导入"按钮
- 拖拽 `.forgify-tool` 文件到工具库界面

### 3.2 导入流程

```
用户选择文件 / 拖拽文件
     ↓
格式校验（版本、必填字段）
     ↓
检查工具名冲突
     ├── 无冲突 → 直接导入
     └── 有冲突 → 弹窗："已有同名工具 [名称]，如何处理？"
                       [重命名] [替换] [取消]
     ↓
导入成功 → 工具出现在工具库（状态为"草稿"，需测试）
```

### 3.3 命名冲突处理

- 重命名：在工具名后追加 `_imported`（如 `send_email_imported`）
- 替换：覆盖现有工具代码（保留 ID 和测试历史不变）

---

## 4. 安全说明

工具是 Python 代码，可以执行任意操作。导入时显示一次性提示：

> "导入的工具包含 Python 代码，运行前请确认来源可信。"

用户点击"我知道了"后不再重复显示（记录到 app_config）。
