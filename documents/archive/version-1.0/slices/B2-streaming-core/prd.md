## 1. 这块做什么

让用户能在 Forgify 里和 AI 进行一次完整的对话，支持流式输出（逐字显示）。这是整个产品的 AI 心脏。

---

## 2. 用户体验

### 2.1 发送消息

- 底部输入框，Enter 发送，Shift+Enter 换行
- 发送后输入框清空，消息出现在对话流中
- 发送时按钮变为"停止"图标

### 2.2 流式响应

- AI 回复逐字出现，有光标闪烁效果
- 生成过程中显示"AI 正在思考..."（首 token 前）
- 收到第一个 token 后光标直接跟随最新文字

### 2.3 停止生成

- 用户点击"停止"：AI 立即停止，已生成的内容保留
- 停止后输入框恢复可用

### 2.4 错误状态

| 错误场景 | 用户看到 |
|---|---|
| 网络超时 | "连接超时，请检查网络后重试" + 重试按钮 |
| API Key 无效 | "API Key 可能已失效，前往设置检查" + 设置按钮 |
| 额度耗尽 | "API 额度不足，请检查账户余额" |
| 模型不可用 | "当前模型不可用，已自动切换到备用模型"（静默降级，只在失败时提示）|

### 2.5 Markdown 渲染

AI 回复支持基本 Markdown 渲染：
- 代码块（带语言高亮）
- 加粗、斜体、行内代码
- 有序/无序列表
- 表格

不支持（简化复杂度）：图片、HTML 嵌入、LaTeX。

## 3. 不在本切片范围内

- 对话的持久化存储（C1）
- 工具调用 / ReAct 循环（ConversationAgent 的工具能力，D 系列）
- 上下文压缩（C4）
- 文件附件（C3）

本切片只做：**单轮纯文本问答，流式显示。**

## 4. 验收标准

- [ ] 发送消息，AI 流式回复，逐字出现
- [ ] 点击停止，生成立即中止，已有内容保留
- [ ] 网络断开时出现正确错误提示
- [ ] API Key 无效时出现正确错误提示
- [ ] Markdown 代码块正常渲染（带高亮）
- [ ] 连续发送多条消息，顺序正确，不串台
