# I3 · Inbox 上下文视图 — 产品需求文档

**切片**：I3  
**状态**：待 Review

---

## 1. 背景

Inbox 消息点击后，右侧不只显示消息详情，还应展示与该消息相关的工作流 Canvas 或工具代码，并高亮出问题所在位置，帮助用户在 Inbox 内直接理解上下文，无需跳转到其他页面。

---

## 2. 上下文视图布局

Inbox 页面的右半区从上到下分为两部分：
- 上方（约 40%）：消息详情 + 操作按钮
- 下方（约 60%）：上下文展示

```
┌────────────────────────────────────────────┐
│  消息详情 + 操作（approve/reject 等）        │
│                                            │
├────────────────────────────────────────────┤
│                                            │
│  工作流 Canvas（只读，失败节点高亮红色）     │
│  或                                        │
│  工具代码（只读，错误行标红）                │
│                                            │
└────────────────────────────────────────────┘
```

---

## 3. 按消息类型展示

| 消息类型 | 上下文视图内容 |
|---|---|
| `approval_required` | 工作流 Canvas，等待审批的节点橙色高亮 |
| `permission_required` | 工具代码（对应工具），权限相关区域高亮 |
| `run_failed` | 工作流 Canvas，失败节点红色高亮，执行路径绿色 |
| `run_completed` | 工作流 Canvas，所有节点绿色（成功状态） |
| `workflow_deployed` | 工作流 Canvas（正常状态） |
| `system` / 其他 | 无上下文视图 |

---

## 4. Canvas 只读模式

Inbox 中展示的 Canvas 是只读的：
- 不可拖拽节点
- 不可添加/删除节点或边
- 不可点击节点配置
- 右上角显示"只读"标识
- 右上角有"打开完整编辑"按钮，点击跳转到资产页面对应的工作流

---

## 5. 节点高亮逻辑

高亮覆盖在节点正常渲染上方：
- 橙色高亮：等待审批
- 红色高亮：运行失败
- 绿色高亮：运行成功
- 灰色：未参与本次运行（跳过）

---

## 6. 验收测试

```
1. 点击 run_failed 消息 → 右侧下方显示工作流 Canvas，失败节点红色高亮
2. 点击 approval_required 消息 → Canvas 显示，等待审批节点橙色高亮
3. Canvas 是只读的：拖拽节点无效，右上角显示"只读"
4. 点击"打开完整编辑"→ 跳转到资产页面对应工作流
5. run_completed 消息 → Canvas 所有节点显示绿色
6. system 类消息 → 无 Canvas，只显示消息内容
```
