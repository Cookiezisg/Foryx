## 1. 这块做什么

用户通过自然语言对话描述需求，AI 生成 Python 工具代码，用户迭代完善后保存为可用工具。
整个过程在对话界面中完成，不需要打开代码编辑器。

---

## 2. 锻造流程

```
用户描述需求
     ↓
AI 理解意图，提出确认问题（如有歧义）
     ↓
AI 生成 Python 函数代码
     ↓
代码出现在对话中（高亮展示）+ "一键测试"按钮
     ↓
用户测试 → 看到结果 → 继续迭代 / 满意后保存
     ↓
工具保存到工具库
```

---

## 3. 用户交互

### 3.1 触发锻造

有两种触发方式：
- 在任意对话中说"帮我做一个工具，能够……"
- 点击工具库的"+ 锻造新工具"按钮（新建锻造对话）

### 3.2 AI 响应格式

AI 在对话中以标准文本 + 代码块的形式回复，代码块用 python 标注语言。  
在代码块下方，对话气泡附带一个操作区：

```
┌────────────────────────────────────────────┐
│ 我为你生成了一个发送邮件的工具。            │
│                                            │
│ ```python                                  │
│ def send_email(to: str, subject: str,      │
│                body: str) -> dict:         │
│     """发送邮件"""                          │
│     ...                                    │
│ ```                                        │
│                                            │
│ [▶ 测试运行]  [💾 保存为工具]              │
└────────────────────────────────────────────┘
```

### 3.3 测试运行

点击"▶ 测试运行" → 弹出参数填写面板：

```
┌─────────────────────────────────┐
│ 测试参数                        │
│                                 │
│ to       [user@example.com    ] │
│ subject  [测试邮件             ] │
│ body     [这是测试内容         ] │
│                                 │
│               [取消] [运行测试] │
└─────────────────────────────────┘
```

运行结果以 `ToolTestResultCard` 卡片展示在对话中（见 C2）：
- 通过：绿色，显示输出
- 失败：红色，显示错误信息

### 3.4 迭代修改

测试失败或用户希望调整 → 直接在对话中说"把超时改成 30 秒"、"增加一个重试参数"等  
AI 生成新版本代码，再次可测试

### 3.5 保存为工具

用户满意后点击"💾 保存为工具"或在对话中说"保存这个工具" → 弹出命名面板：

```
┌──────────────────────────────────┐
│ 保存工具                         │
│                                  │
│ 名称     [发送邮件              ] │
│ 描述     [通过 SMTP 发送邮件    ] │
│ 分类     [邮件            ▼    ] │
│                                  │
│                  [取消] [保存]   │
└──────────────────────────────────┘
```

保存后：对话中出现 `ToolCreatedCard` 卡片，工具出现在工具库。

## 4. 工具规格约束

AI 生成的工具代码必须满足：
- 顶部必须有 4 个 `# @` 元数据标注（version, category, display_name, description）
- 只有一个顶层函数，函数名即工具名（snake_case）
- 参数必须有类型注解（`str`、`int`、`float`、`bool`、`list`、`dict`），支持泛型如 `list[int]`
- 返回值类型为 `dict`
- 有 docstring 说明功能
- 可以使用 `import`（依赖会被自动安装）

示例：
```python
# @version 1.0
# @category email
# @display_name 发送邮件
# @description 通过 SMTP 发送邮件到指定地址

def send_email(to: str, subject: str, body: str) -> dict:
    """通过 SMTP 发送邮件到指定地址"""
    import smtplib
    ...
    return {"success": True, "message": "邮件已发送"}
```

**元数据标注说明：**
- `@version` — 语义版本号，创建时为 1.0
- `@category` — 枚举值：email/data/web/file/system/other
- `@display_name` — 中文名，不超过 10 字
- `@description` — 一句话描述，不超过 30 字
- 用户创建的工具自动加 `# @custom`，内置工具加 `# @builtin`
- 后端 `NormalizeCodeAnnotations()` 确保标注始终与 DB 字段一致

## 5. 错误情况

| 情况 | 处理 |
|---|---|
| 代码语法错误 | 错误信息展示在 TestResult 卡片，AI 自动修复 |
| 依赖安装失败 | 提示具体包名，AI 建议替代方案 |
| 运行超时（>30s） | 提示超时，AI 建议优化 |
| 用户描述的功能不可实现 | AI 解释限制，提出可行替代方案 |
