# C3 · 文件附件 — 产品需求文档

**切片**：C3  
**状态**：待 Review  
**依赖**：A2、C1、C2  
**下游**：D2（工具锻造时上传样例文件）

---

## 1. 这块做什么

用户可以把文件拖入对话，Forgify 读取文件内容并注入 AI 上下文，让 AI 能"看见"文件内容。

---

## 2. 支持的文件类型

| 类型 | 格式 | 处理方式 |
|---|---|---|
| 纯文本 | .txt、.md、.csv、.json、.yaml、.xml、.log | 直接读取文本内容 |
| 代码 | .py、.js、.ts、.go、.sql 等 | 直接读取，保留格式 |
| Excel | .xlsx、.xls | 转换为 Markdown 表格（每个 Sheet 一个表格） |
| PDF | .pdf | 提取文本内容（不含图片） |
| Word | .docx | 提取文本内容 |
| 图片 | .png、.jpg、.gif、.webp | 直接传给支持视觉的模型（如 Claude） |

**不支持**：.zip、.exe、.mp4 等二进制或多媒体文件（提示用户不支持）

---

## 3. 交互流程

### 3.1 拖拽上传

- 拖拽文件到对话区域 → 出现蓝色虚线边框高亮区
- 松开 → 文件出现在输入框上方的"附件区"
- 支持同时拖入多个文件

### 3.2 点击上传

- 输入框工具栏有"📎"图标
- 点击打开系统文件选择器

### 3.3 附件预览（发送前）

文件出现在输入框上方：

```
┌──────────────────────────────────────┐
│  📄 供应商报价.xlsx  ×               │
│  📄 邮件模板.docx   ×               │
└──────────────────────────────────────┘
│  发送消息...                [Send]   │
└──────────────────────────────────────┘
```

- 每个附件有删除按钮（×）
- 文件名过长则截断显示

### 3.4 发送后展示

消息气泡内显示文件附件：

```
│  请帮我分析这个报价表                │
│  📄 供应商报价.xlsx (15KB)           │
```

图片附件直接显示缩略图。

### 3.5 文件内容注入

发送时，Forgify 在消息前追加文件内容：

```
[附件: 供应商报价.xlsx]
| 供应商 | 产品 | 单价 | 数量 |
|--------|------|------|------|
| 供应商A | 螺丝 | 0.5 | 1000 |
...
[附件结束]

用户消息：请帮我分析这个报价表
```

图片则直接作为多模态内容传给 LLM（不转文字）。

---

## 4. 限制

- 单文件最大：20MB
- 单次对话最多同时附：5 个文件
- Excel/PDF 超过 100 页/行时，只取前 100 页/行并提示"文件过大，已截取前 100 行"

---

## 5. 错误处理

| 情况 | 提示 |
|---|---|
| 不支持的文件类型 | "不支持 .zip 文件，请使用文本或 Office 文件" |
| 文件过大 | "文件超过 20MB 限制" |
| 读取失败 | "无法读取文件，请确认文件未损坏" |
| 模型不支持图片 | "当前模型不支持图片，已提取文件名作为描述" |

---

## 6. 验收标准

- [ ] 拖入 .txt 文件，AI 能正确引用文件内容回答问题
- [ ] 拖入 .xlsx 文件，AI 看到的是 Markdown 表格
- [ ] 拖入 .png 图片，传给支持视觉的模型（Claude/GPT-4o）
- [ ] 不支持的文件类型显示正确错误
- [ ] 超过 20MB 的文件被拒绝
- [ ] 发送后消息中显示文件名和大小
