# J1 · Home Page — 产品需求文档

**切片**：J1  
**状态**：待 Review

---

## 1. 背景

侧边栏第一个标签页是 Home Page。目标是让用户打开应用后能快速看到整体状态，并立即开始对话。

---

## 2. 布局

```
左侧面板（320px）            右侧主区
─────────────────────────  ─────────────────────────────
  状态摘要                   [大型对话输入区]
  ─────────────
  今日已运行 3 次             和普通对话页一样的 AI 对话界面
  2 个工作流已部署            用户可以直接在这里开始对话
  1 个待审批消息              这里开启的对话默认不绑定任何资产
  ─────────────
  最近活动
  ─────────────
  · 发送周报邮件 ✅ 刚刚
  · 数据处理 ❌ 5分钟前
  · 发送周报邮件 ✅ 昨天
```

---

## 3. 左侧面板内容

### 状态摘要（Stats）

- 今日工作流运行次数
- 已部署工作流数量
- 待审批消息数量（点击跳转到 Inbox）

### 最近活动（Recent Activity）

列出最近 10 条工作流运行记录（跨所有工作流），每条显示：
- 工作流名称
- 状态图标（✅/❌/⏳）
- 相对时间

点击记录跳转到对应工作流的资产页面，并打开运行历史。

---

## 4. 右侧：对话区

与对话页面（C 系列）复用同一个 ChatInterface 组件。

- 这里开启的对话默认不绑定任何工作流或工具（普通对话）
- 如果聊天中 AI 创建了工作流，则自动绑定（E2 流程），同时对话页面会出现该对话
- 占位提示：`和 Forgify 助手聊聊，告诉我你想自动化什么...`

---

## 5. 验收测试

```
1. 打开应用 → Home Page 显示今日运行次数、已部署数量、待审批数量
2. 待审批数量显示红色，点击跳转到 Inbox
3. 最近活动列表显示最新 10 条运行记录
4. 点击活动记录 → 跳转到对应工作流资产页面
5. 右侧对话区可正常发送消息，收到 AI 回复
6. Home Page 对话中创建工作流 → 自动绑定，对话出现在对话列表中
```
