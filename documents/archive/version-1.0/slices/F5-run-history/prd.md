# F5 · 运行历史 — 产品需求文档

**切片**：F5  
**状态**：待 Review

---

## 1. 背景

每次工作流运行（手动/定时/事件触发）都应被记录。用户需要能够查看历史运行记录、了解每次运行的执行路径和节点结果。

---

## 2. 入口

工作流 Canvas 顶部工具栏有"运行历史"按钮，点击后在 Canvas 下方展开运行历史面板（或切换到历史标签页）。

---

## 3. 运行列表

```
运行历史
────────────────────────────────────────────────
  ✅ 手动运行     2026-04-19 14:32   3.2s
  ❌ 定时运行     2026-04-19 09:01   1.8s  节点"发送邮件"失败
  ✅ 手动运行     2026-04-18 22:15   5.1s
  ✅ 定时运行     2026-04-18 09:00   2.9s
────────────────────────────────────────────────
```

每条记录显示：
- 状态图标（✅/❌/⏳）
- 触发方式（手动/定时/Webhook/文件）
- 时间
- 总耗时
- 失败时：失败节点名称

---

## 4. 运行详情

点击一条历史记录，Canvas 切换为"历史回放"模式：
- 节点颜色显示该次运行的最终状态（绿色/红色/灰色跳过）
- 执行路径：已执行的边高亮显示
- 点击任意节点查看该节点在本次运行的输入/输出

历史回放是只读的，不可修改节点配置。

---

## 5. 清理策略

- 默认保留最近 **100 条** 运行记录
- 超出时自动删除最旧的记录
- 用户可在设置中调整保留数量（20 / 50 / 100 / 不限制）

---

## 6. 验收测试

```
1. 手动运行完成后 → 运行历史列表出现新记录
2. 点击历史记录 → Canvas 切换为历史回放模式，节点颜色反映该次状态
3. 点击历史回放中的节点 → 显示该次运行的输入/输出
4. 失败的运行记录 → 显示失败节点名称
5. 超过 100 条时 → 最旧的自动删除
```
