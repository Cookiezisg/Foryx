## 1. 循环节点 (Loop)

对列表中的每一项执行相同的子流程。

```
[循环节点] ←────────────────────┐
    配置：输入列表 = {{emails}}   │
    │                             │
    ▼                             │
[子流程节点 A] → [子流程节点 B] ──┘
    │（每项处理完继续下一项）
    ▼（所有项处理完后）
[下游节点]
```

配置：
- 迭代列表：绑定上游的数组输出
- 并发数：1（顺序）/ N（并行执行 N 个）
- 当前项变量名（`item`，在子流程中用 `{{item.field}}`）
- 输出：所有项的结果组成的数组

---

## 2. 并行节点 (Parallel)

同时执行多个独立分支，等待所有分支完成后汇合。

```
          [并行节点]
         /     |     \
[分支A] [分支B] [分支C]
         \     |     /
          [汇合节点]
```

配置：
- 分支数量（手动添加）
- 汇合策略：等待全部完成 / 等待第一个完成
- 某分支失败时：停止其他分支 / 忽略继续

---

## 3. 子工作流节点 (Sub-workflow)

将另一个工作流作为节点嵌入当前工作流。

配置：
- 选择已有工作流
- 传入参数映射（当前节点输出 → 子工作流触发参数）
- 输出：子工作流的最终输出

用途：避免重复构建相同逻辑，实现工作流复用。

---

## 4. 验收标准

- [ ] 循环节点：传入含 3 个邮件的列表 → 对每个邮件执行子流程 → 返回 3 个结果
- [ ] 并行节点：两个分支同时执行，执行时间约等于最慢分支的时间
- [ ] 子工作流节点：调用另一个 deployed 工作流并传参
