## 1. 这块做什么

对话中间栏的完整消息流体验：消息的展示、持久化、各类消息卡片的渲染。

---

## 2. 消息类型

### 2.1 用户消息

- 右对齐气泡
- 支持多行文本
- 有附件时显示文件缩略图/图标（C3 负责）

### 2.2 AI 文本回复

- 左对齐，无气泡（直接在背景上渲染）
- Markdown 渲染（B2 已实现）
- 流式时有光标，完成后光标消失
- 左上角显示模型名称（小字，灰色）

### 2.3 AI 操作卡片

当 AI 创建/修改了工具或工作流，在对话中显示一张卡片（非纯文本）：

```

│  状态：draft · 等待测试      │
│  [打开工具详情]              │
└─────────────────────────────┘
```

```

│  变化：添加了"汇总"节点      │

```

## 3. 滚动行为

- **自动跟随**：有新消息或 AI 正在流式输出时，自动滚动到底部
- **用户上滚打断**：用户向上滚动时，停止自动跟随；出现"↓ 回到最新"按钮
- **点击回到最新**：恢复自动跟随，跳到底部
- 初次进入对话：直接滚到底部（无动画）

---

## 4. 消息持久化

- 每条消息（用户发送 + AI 完整回复）存入数据库
- AI 流式输出期间不存储（仅在 `chat.done` 事件后一次性保存完整内容）
- 重新打开对话：从数据库加载历史消息，渲染历史记录

---

## 5. 空白状态

新建对话进入时，中间区域显示：

```
         Forgify
   
   开始一段对话，或拖入文件
   
   [/ 指令]  [@ 引用]
```

无其他引导内容，保持简洁。
