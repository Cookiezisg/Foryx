# I2 · 审批工作流 — 产品需求文档

**切片**：I2  
**状态**：待 Review

---

## 1. 背景

Inbox 中的审批消息（来自 F3 Approval 节点和 H1 权限门控）需要用户做出决策。I2 定义审批消息的详情视图和操作交互。

---

## 2. 审批消息详情

在 Inbox 右侧详情区显示：

```
⏳ 需要你的确认
───────────────────────────────
工作流：发送周报邮件
节点：邮件发送确认

即将发送邮件至 boss@company.com
主题：本周工作汇报
正文预览：...

参数
  to:      boss@company.com
  subject: 本周工作汇报
  body:    ...

                    [拒绝]  [修改后批准]  [批准]
```

---

## 3. 修改后批准

点击"修改后批准"时：
- 参数区切换为可编辑模式
- 用户修改任意参数值
- 点击"确认修改并批准"提交

---

## 4. 审批完成后

- 审批消息状态变为 `resolved`
- 右侧详情显示"已批准"或"已拒绝"的结果徽章
- 工作流在 Canvas 上继续执行（实时更新）

---

## 5. 权限确认消息

来自 H1 的权限确认消息格式略有不同：

```
🔐 权限确认
───────────────────────────────
工具 gmail_send 需要执行权限

即将：发送邮件
参数：{"to":"...","subject":"..."}

                    [拒绝]  [修改后批准]  [始终允许]
```

"始终允许"操作调用 `PermissionService.Grant()`，之后该工具不再需要确认。

---

## 6. 验收测试

```
1. approval_required 消息 → 详情显示标题、消息内容、参数
2. 点击"批准" → ApproveAction() → 消息 resolved → 工作流继续
3. 点击"拒绝" → RejectAction() → 消息 resolved → 工作流按 reject_mode 处理
4. 点击"修改后批准" → 参数可编辑 → 修改后 ApproveWithModifiedParams()
5. permission_required 消息 → "始终允许"按钮 → Grant() → 消息 resolved
6. 审批完成后右侧显示结果徽章
```
