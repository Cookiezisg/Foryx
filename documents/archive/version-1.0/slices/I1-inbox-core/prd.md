# I1 · Inbox 核心 — 产品需求文档

**切片**：I1  
**状态**：待 Review

---

## 1. 背景

Inbox 是侧边栏的第四个标签页，展示所有来自工作流的通知：审批请求、运行失败、部署成功等。

---

## 2. 消息列表

Inbox 左侧是消息列表：

```
Inbox                           [全部已读]
────────────────────────────────────────
  🔴 ⏳ 需要确认 · 发送周报邮件         刚刚
  🟡 ❌ 工作流失败 · 数据处理           5分钟前
  ⚪ ✅ 工作流已部署 · 邮件提醒          昨天
  ⚪ ✅ 运行成功 · 发送周报邮件          昨天
────────────────────────────────────────
```

- 未读消息（pending）左侧显示红点
- 高优先级消息（审批）排在最前
- 点击消息：标记已读，右侧显示详情（I3）

---

## 3. 侧边栏图标

侧边栏 Inbox 图标上方显示红色未读数量徽章（pending 消息数），最多显示 99+。

---

## 4. 消息分组

消息按时间分组：
- 今天
- 昨天
- 更早

---

## 5. 操作

- **全部已读**：将所有 pending 消息标记为 read
- **单条关闭**：点击消息右侧的 ×，dismiss 该条消息
- 已 resolved 的消息显示在列表末尾，不影响未读计数

---

## 6. 验收测试

```
1. 工作流运行失败 → Inbox 出现新消息 → 侧边栏图标显示红点
2. 点击消息 → 消息标记已读 → 红点消失（无其他未读时）
3. 点击"全部已读" → 所有 pending → read
4. 点击 × → 消息 dismissed，从列表消失
5. 消息按优先级和时间正确排序
6. 今天/昨天/更早分组正确显示
```
