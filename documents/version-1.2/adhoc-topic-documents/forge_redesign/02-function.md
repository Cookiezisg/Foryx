## 1. 定位

Function 是 Trinity 中**最轻**的产物 —— stateless 一次性 Python 函数,**跑完即销**。

| 属性 | 值 |
|---|---|
| 状态 | 无(每次跑新 subprocess) |
| 调用频次 | 一次性 / 反复都 OK |
| Lifetime | 单次调用(execution-scoped) |
| 形态 | LLM 写的 Python 函数 |
| 承载 | sandbox v2 Python EnvManager + per-call temp owner |

### 与现有 forge 的关系
**完全替代,推倒重做**(决策见 [`00-overview.md`](./00-overview.md) §2.1)。新 entity / table / API / LLM tool 全套独立设计。无数据迁移路径(forge 是开发期数据,V1 上线前清空)。

## 2. 锻造 / 调用模型

### 2.1 锻造 — `create_function` / `edit_function` ops 流式

LLM 通过 ops 流式构建 Definition。每个 op 应用 emit 一个 progress delta(详见 [`01-shared-tool-interface.md`](./01-shared-tool-interface.md) §4.2)。

Definition 包含:
- `name` / `description` / `tags`
- `code`(Python 函数主体)
- `parameters`(JSON schema,入参约束)
- `returnSchema`(JSON schema,返回约束)
- `dependencies`(PEP 508 specifier list)
- `pythonVersion`(PEP 440 spec,可空走默认 `>=3.12`)

**首次创建 + 校验通过 → auto-accept**(对齐 forge TE-15)。

### 2.2 调用 — `run_function`

每次调用:
1. Resolve active version
2. EnsureEnv(owner = 临时 ephemeral owner = `function-call:<callId>`)
3. Spawn Python subprocess(driver 模板 + user code)
4. 通过 stdin 喂 args JSON
5. stdout 读 result JSON
6. 30s 默认超时(`config.timeout` 可 override)
7. **Subprocess 跑完即销**;ephemeral owner 在调用结束时 destroy env(可选 — 看是否需要 per-call 隔离 vs 共享 venv)

实际实现:**共享 venv**(per Definition 的 env_id),**不 per-call 起新 venv**。原因:同 Function 反复调用时每次重建 venv 极慢。共享 venv 安全因为 Python subprocess 隔离已足够。

## 3. Op 集合

| Op | 字段 | 校验 |
|---|---|---|
| `set_meta` | `name?, description?, tags?` | name 非空(create) / partial UNIQUE |
| `set_code` | `code: string` | 非空,Python AST 可解析,函数名跟 name 匹配 |
| `set_parameters` | `parameters: ParameterSpec[]` | 每条 schema 合法,name 唯一 |
| `set_return_schema` | `returnSchema: object` | JSON Schema 合法 |
| `set_dependencies` | `deps: string[]` | 每条 PEP 508 解析合法 |
| `set_python_version` | `version: string` | PEP 440 解析合法,在 mise 支持范围 |

### 3.1 ParameterSpec

```json
{
  "name": "text",
  "type": "string",      // string | number | integer | boolean | object | array
  "description": "Input text to process",
  "required": true,
  "default": null,
  "enum": null            // 可选 enum 约束
}
```

## 4. Python 代码契约

锻造时 LLM 写出的 Python 函数必须满足:

1. **函数名跟 Function `name` 一致**(driver 自动 import 调用)
2. **单个公开函数**(其他辅助函数允许私有 `_`-prefixed)
3. **入参跟 `parameters` spec 对齐**(driver 按 schema 装载 args 调用)
4. **返回 JSON-serializable**(driver 序列化输出)
5. **不能 import Handler client**(决策 D7)— 校验时静态扫 import 黑名单
6. **PathGuard 守敏感路径**(沙箱 v2 现有约束)

### 4.1 driver 模板

```python
# Auto-generated by Forgify; do not edit.
import sys, json, traceback
sys.path.insert(0, '/sandbox/lib')

from user_function import {function_name}  # noqa: E402

def main():
    args = json.loads(sys.stdin.read())
    try:
        result = {function_name}(**args)
        sys.stdout.write(json.dumps({"ok": True, "output": result}))
    except Exception as e:
        sys.stdout.write(json.dumps({
            "ok": False,
            "error": str(e),
            "trace": traceback.format_exc()
        }))

if __name__ == "__main__":
    main()
```

### 4.2 用户函数示例

```python
# user_function.py — 由 LLM 锻造
def markdown_to_pdf(text: str, title: str = "Document") -> dict:
    """Convert markdown to PDF, return {pdf_path, page_count}."""
    import markdown, weasyprint
    html = markdown.markdown(text)
    full_html = f"<h1>{title}</h1>{html}"
    pdf_bytes = weasyprint.HTML(string=full_html).write_pdf()
    
    import tempfile, os
    fd, path = tempfile.mkstemp(suffix=".pdf")
    os.write(fd, pdf_bytes)
    os.close(fd)
    
    return {"pdf_path": path, "page_count": 1, "size_bytes": len(pdf_bytes)}
```

## 6. Function 不能调 Handler — D7 决策

### 决策
**Function 内 Python 代码不允许 import / 调用 Handler client**。

### 理由
- Function 是纯叶子节点(stateless),组合靠 workflow 节点编排
- 防递归 / 防循环依赖
- 强制可视编排(用户能从 workflow 图上看到 function ↔ handler 的调用关系,不藏在 Python 里)
- 跟设计哲学一致(D5 各 domain 隔离)

### 实施
- **静态扫 import 黑名单**:`set_code` op 应用时扫 import 列表,出现 `from forgify_handler import` 类的语句直接 reject
- **运行时拦截**:driver 运行时不注入任何 Handler client(没法 import 起)

### 用户场景如何替代
- "Function 想用数据库" → 不写 SQL 函数,而是 workflow 节点 `handler.pg.query` + `function.process` 串起来
- "Function 想发 HTTP" → workflow 节点 `http` + `function.parse` 串起来,或 function 内自己用 Python `requests`(不是 Handler,是 stdlib / 第三方库 — 允许)

## 7. LLM 工具集

```typescript
search_function({ query?, limit?, cursor? })
  → { items: FunctionSummary[], nextCursor?, hasMore }

get_function({ id })
  → { function: Function, activeVersion?: FunctionVersion, pending?: FunctionVersion }

create_function({ name, description, ops, changeReason })
  → { id, version, status, opsApplied }
  // 流式:每 op emit progress delta + env sync 期间 emit progress

edit_function({ id, ops, changeReason })
  → { pendingId, opsApplied }
  // 流式:同 create

revert_function({ id, targetVersion })
  → { pendingId }

delete_function({ id })
  → { deleted }

run_function({ functionId, args, version? })
  → { ok, output, error?, elapsedMs }
  // 流式:env sync 进度 + Python stdout/stderr 行 emit progress
```

详见 [`01-shared-tool-interface.md`](./01-shared-tool-interface.md) §2 / §4。

## 9. 错误码

| Code | HTTP | Sentinel | 触发 |
|---|---|---|---|
| `FUNCTION_NOT_FOUND` | 404 | `function.ErrNotFound` | id 查不到 |
| `FUNCTION_NAME_DUPLICATE` | 409 | `function.ErrDuplicateName` | 重名 |
| `FUNCTION_VERSION_NOT_FOUND` | 404 | `function.ErrVersionNotFound` | revert/get version 版本不在 |
| `FUNCTION_PENDING_NOT_FOUND` | 404 | `function.ErrPendingNotFound` | accept/reject 无 pending |
| `FUNCTION_PENDING_CONFLICT` | 409 | `function.ErrPendingConflict` | edit 时已有 pending |
| `FUNCTION_RUN_FAILED` | 422 | `function.ErrRunFailed` | sandbox 内部错(≠ ok=false) |
| `FUNCTION_AST_PARSE_FAILED` | 422 | `function.ErrASTParseError` | 代码 AST 不通过 |
| `FUNCTION_NO_ACTIVE_VERSION` | 422 | `function.ErrNoActiveVersion` | 无 active(草稿 + pending 未 accept) |
| `FUNCTION_ENV_NOT_READY` | 422 | `function.ErrEnvNotReady` | env status=syncing/evicted/pending |
| `FUNCTION_ENV_FAILED` | 422 | `function.ErrEnvFailed` | env sync 失败 |
| `FUNCTION_DEPENDENCY_RESOLUTION` | 422 | `function.ErrDependencyResolution` | uv 解析依赖失败 |
| `FUNCTION_SANDBOX_UNAVAILABLE` | 503 | `function.ErrSandboxUnavailable` | sandbox bootstrap 失败 |
| `FUNCTION_OP_INVALID` | 400 | `function.ErrOpInvalid` | ops 应用失败(详细 op index 在 message) |

## 10. Sandbox 集成

每个 FunctionVersion 绑一个 venv:
- env_id = sha256(deps + python_version)
- 同 deps + Python 的多 Function 共享 venv(uv hardlink wheel 缓存让磁盘 ≈ 1×)
- EnvManager=python(sandbox v2 现有)
- Spawn 时调 `sandboxapp.Service.Spawn(owner, ...)`,owner = `function-call:<callId>`

env sync 在 create / edit 时同步等完(对齐 forge 现状),sync 进度通过 progress block emit。

## 11. Catalog source

```go
// app/function/catalog_source.go
func (s *Service) AsCatalogSource() catalogdomain.CatalogSource {
    return &catalogSource{svc: s}
}

func (cs *catalogSource) Name() string { return "function" }
func (cs *catalogSource) Granularity() catalogdomain.Granularity { return catalogdomain.PerItem }
func (cs *catalogSource) ListItems(ctx context.Context) ([]catalogdomain.Item, error) {
    fns, _ := cs.svc.List(ctx, ...) // active + non-deleted
    items := make([]catalogdomain.Item, 0, len(fns))
    for _, fn := range fns {
        items = append(items, catalogdomain.Item{
            Source:      "function",
            Name:        fn.Name,
            Description: fn.Description,
            Category:    "computation", // optional
        })
    }
    return items, nil
}
```

main.go 装配:`catalogService.RegisterSource(functionService.AsCatalogSource())`。
